LandXML, DXF & CAD exports for solar sites

From KMZ boundary to a civil-ready starter package — screening-grade, download immediately after each run.

What’s in the ZIP

  • Georeferenced LandXML — TIN surface in UTM (WGS84) with coordinate system metadata
  • DXF contours — major/minor contour layers; parcel linework on SITE_BOUNDARY
  • Local-centroid package — same surface and linework with origin at site centroid for quick layout
  • Reference JSON — EPSG, units, import notes, and project metadata
  • CSV / XYZ — optional point clouds for GIS workflows

Parcel linework

Site boundaries from your KMZ or drawn polygons export as linework — not as a separate “boundary-only” DXF. In CAD, look for layer SITE_BOUNDARY on the contour DXF files and in LandXML Parcels / PlanFeatures.

USA projects — US Survey Feet

When the project country is United States, georeferenced LandXML and DXF use US Survey Feet. Import into an imperial CAD drawing without applying extra feet↔meter scaling.

Recommended workflow

  1. Import georef LandXML as the primary surface (or local package for quick sketch layout)
  2. Overlay SITE_BOUNDARY linework for parcel context
  3. Use contours for grading dialogue — not as final design contours
  4. Replace or supplement with LiDAR / RTK before detailed civil design
TopoIQ does not store exports server-side. Download your ZIP immediately after each analysis run.
