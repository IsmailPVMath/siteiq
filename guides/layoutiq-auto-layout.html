LayoutIQ — auto layout on your buildable area

Pick mount type, pitch, and GCR — PVMath packs rows inside GIS setbacks and feeds YieldIQ automatically.

What LayoutIQ does

LayoutIQ turns your site boundary (or SiteIQ buildable polygon) into a packed row layout for early capacity and energy screening. You choose:

  • Mount type — Fixed Tilt or Single-Axis Tracker (chosen here, not in Project Setup)
  • Portrait count — 1P–4P fixed or 1P/2P tracker
  • Pitch & GCR — row spacing and ground coverage
  • Optimization mode — balanced, high-energy, land-optimized, or custom

GIS setbacks applied

When SiteIQ GIS analysis has run, LayoutIQ can pack rows on the buildable area — roads, buildings, water, and other OSM-detected constraints are already buffered. You can also use the full boundary if you prefer to ignore exclusions for an early sketch.

Key outputs

  • DC capacity & module count — from packed rows on the polygon
  • Layout sweep table — compare mount/portrait/GCR combinations
  • 3D terrain + layout preview — rows on the TerrainIQ DEM surface
  • DXF export — row linework for CAD overlay (screening-grade)
  • BOM CSV — module and row counts in the project ZIP

Feeds YieldIQ

YieldIQ runs on your selected LayoutIQ configuration — mount type, GCR, and losses — so energy results match the layout you picked, not a generic four-config comparison table.

Workflow order: Project Setup → SiteIQ → TerrainIQ → LayoutIQ → YieldIQ → PVMath Report / ZIP. One login at app.pvmath.com.
